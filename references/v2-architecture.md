# V2.1 Lore Engine 架构

> 来源: video-to-md/prompt_builder_v2.py

V2 架构借鉴 Lore Engine 的分层提示词设计，解决 V1 "流水账"问题。
V2.1 增强知识补全深度、讲课顺序结构、Gemini 视频分析深度。

---

## 核心改进

| 问题 | V1 | V2.1 |
|------|----|------|
| 组织方式 | 按时间点（138个章节）| lecture: 跟随讲课顺序；其他: 按知识结构 |
| 截图处理 | 每个截图一个章节 | 截图分配到知识点下 + 深度分析（OCR/图表/多帧） |
| 内容深度 | 单一风格 | 三种深度可选 |
| 知识补全 | 无 | 五层补全（术语→推导→演化→应用→纠错）|
| 变化检测 | 10-30字描述 | 30-100字含 OCR/图表结构/公式 |
| 伪代码 | ```python 标注 | 无语言标注 + 后处理兜底 |

## 分层架构（v2.1 优化后组装顺序）

```
┌─────────────────────────────────────┐
│       BASE_INSTRUCTIONS             │  基础指令（角色、语言、格式、代码块卫生、伪代码规则）
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│       CONTENT_MODES                 │  内容类型模式
│  video_tutorial | lecture | code    │
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│       DEPTH_LEVELS                  │  深度控制
│  short_hand | balanced | deep_dive  │
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│    STRUCTURE (模式依赖)             │  组织结构
│  lecture → LECTURE_STRUCTURE         │  跟随讲课顺序
│  其他   → HIERARCHICAL_STRUCTURE    │  按知识领域
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│    CONTINUITY_RULES (分段时)        │  分段连贯性（chunk>0 时注入）
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│    TRANSCRIPT_INTEGRATION           │  音频转录（主干，优先于截图）
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│    INTELLIGENT_INFERENCE            │  五层智能补全
│  术语→推导→演化→应用→纠错          │
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│    SCREENSHOT_INTEGRATION           │  截图集成 + 深度分析
│  含 OCR / 图表理解 / 多帧关联      │  （辅助，在推理之后）
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│         AI_SUMMARY                  │  AI 摘要
└─────────────────────────────────────┘
                ↓
┌─────────────────────────────────────┐
│    TOOLS (lecture 模式)             │  LaTeX + 表格 + 理解题
└─────────────────────────────────────┘
```

> **关键顺序变化**: 转录（音频主干）在截图之前注入，
> 确保 LLM 以音频为主干、截图为辅助（双通道融合: 音频 > 视觉）

## 组织结构策略 (v2.1 新增区分)

### LECTURE_STRUCTURE（讲座模式）

```
大结构：跟随讲课顺序
- 教授先讲 A 再讲 B → 笔记中 A 在 B 前面
- 幻灯片切换顺序 = 章节顺序

小结构：知识逻辑组织
- 同一概念的多次提及归并
- 穿插例子放到对应概念下
- 目标 5-10 个 ## 章节
```

### HIERARCHICAL_STRUCTURE（教程/代码模式）

```
按知识领域划分（不按时间顺序）
- 核心功能、插件系统、使用技巧等
- 子主题 → 知识点 → 截图
```

## 内容类型模式 (CONTENT_MODES)

### lecture（讲座/演示）— v2.1 增强

```
特点：
- 幻灯片驱动，教授口述远比幻灯片丰富
- 口语化表达需转为正式书面体
- 概念渐进式构建

处理要求：
- 口语→书面转换，忽略填充词
- 完整推理链，形式化定义
- 算法伪代码（不标注语言）+ 时间/空间复杂度
- 数学公式用 LaTeX
- 主动对比（表格列出关键差异）
- 易错点标注（⚠️ 格式高亮）
- 知识补全（五层补全体系）
- 自动启用 TOOLS: LaTeX + 表格 + 理解题
- 使用 LECTURE_STRUCTURE（跟随讲课顺序）
```

### video_tutorial（视频教程）

```
特点：
- 大量操作演示和界面切换
- 讲解者边操作边解释

处理要求：
- 合并操作序列为功能说明
- 解释"为什么这样做"
- 使用 HIERARCHICAL_STRUCTURE（按知识领域）
```

### code_demo（编程教程）

```
特点：
- 代码编写和运行、调试
- 技术概念解释

处理要求：
- 保留完整代码示例
- 解释代码原理
- 使用 HIERARCHICAL_STRUCTURE（按知识领域）
```

## 智能补全 (INTELLIGENT_INFERENCE) — v2.1 五层体系

| 层次 | 补全内容 | 示例 |
|------|---------|------|
| 1. 术语定义 | 首次出现的技术术语 | "双向链接" → 定义 + 重要性 |
| 2. 推导与证明 | 关键定理/性质的推导 | "UCS 最优" → 路径代价单调递增证明 |
| 3. 概念演化链 | 概念之间的因果关系 | DFS缺陷 → BFS → BFS局限 → UCS |
| 4. 应用指导 | 选择建议和退化场景 | "什么条件用 DFS 而不是 BFS？" |
| 5. 纠错补充 | 修正不严谨表述 | `> 📌 **补充**` 格式 |

## 截图深度分析 (v2.1 新增)

Stage 3 收到截图图片后，必须执行：

1. **文字 OCR**：抄录幻灯片标题、要点、公式
2. **图表理解**：描述搜索树、状态图、流程图的结构和含义
3. **代码转录**：截图中的代码转为 code block
4. **多帧关联**：相邻截图描述演变过程（状态A → B → C）

---

## 变化检测增强 (v2.1)

Stage 2 的 CHANGE_DETECTION_PROMPT 要求 description 包含：
- 幻灯片标题和要点（原文抄写）
- 公式（LaTeX 格式）
- 图表结构描述
- 长度：30-100 字（旧版 10-30 字）

## 后处理管道 (v2.1)

```
后处理 1：修复反引号包裹的图片引用
后处理 2：验证图片引用，移除不存在的文件
后处理 3：修复未关闭的代码块
后处理 4：压缩连续 3+ 空行
后处理 4.5：修复伪代码语言标注（```python → ```）
后处理 5：Media Extended 时间戳转换
后处理 6：视频嵌入
```

## 深度控制 (DEPTH_LEVELS)

### short_hand（极简模式）
- 要点列表，核心术语和关系，最大信息密度

### balanced（平衡模式，推荐）
- 段落+列表混合，清晰解释，术语分级，纠错标注

### deep_dive（深度模式）
- 全面解释，丰富示例，边界情况，思考题

## PromptBuilderV2 使用

```python
# 讲座视频配置（v2.1 推荐）
prompt = (PromptBuilderV2()
    .with_mode("lecture")        # → LECTURE_STRUCTURE
    .with_depth("balanced")
    .with_hierarchy()            # 自动选择 LECTURE_STRUCTURE
    .with_transcript(transcript)
    .with_inference()            # 五层补全
    .with_screenshots(screenshots)  # 含深度分析指令
    .with_summary()
    .with_latex()
    .with_tables()
    .with_tricky_questions()
    .build())

# 教程视频配置
prompt = (PromptBuilderV2()
    .with_mode("video_tutorial")  # → HIERARCHICAL_STRUCTURE
    .with_depth("balanced")
    .with_hierarchy()
    .with_transcript(transcript)
    .with_inference()
    .with_screenshots(screenshots)
    .with_summary()
    .build())
```

## 与 V1 的关键区别

| 维度 | V1 (BiliNote) | V2.1 (Lore Engine) |
|------|--------------|---------------------|
| 组织逻辑 | 按时间顺序 | lecture: 讲课顺序 / 其他: 知识结构 |
| 截图处理 | 每个截图一章 | 语义分组 + OCR/图表深度分析 |
| 内容深度 | 9种风格（cosmetic）| 3种深度（structural）+ 五层补全 |
| 知识补全 | 无 | 术语→推导→演化→应用→纠错 |
| 变化检测 | 简短描述 | 详细 OCR + 图表结构 |
| 伪代码 | 可能标注 python | 无语言标注 + 后处理兜底 |
| 适用场景 | 快速记录 | 教学材料 |
